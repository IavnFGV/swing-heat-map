# Swing Heat Map — монтажный проект без героизма

Цель: собирать первый ролик параллельно с исследованием и не возвращаться позднее к горе несортированного материала.

Это editor-agnostic монтажный лист. Его можно использовать в DaVinci Resolve, Premiere, CapCut, Kdenlive или другом редакторе.

---

## Минимальный стандарт качества

Для первого выпуска достаточно:

- ясной последовательности событий;
- читаемого кода;
- разборчивого голоса;
- отсутствия длинных пауз;
- одинаковой громкости;
- 3–5 простых текстовых подписей;
- подтверждённых технических утверждений.

Не нужны:

- анимационные переходы;
- motion design;
- камера;
- музыка;
- звуковые эффекты;
- сложная цветокоррекция;
- идеальная дикция;
- десятки быстрых склеек.

Если выбор стоит между эффектом и понятным объяснением, выбирается объяснение.


## Черновая сборка прямо сейчас

На timeline уже можно положить:

### Блок A — современный результат

Источник:

```text
artifacts/current/03-current-ui.mp4
```

Действие:

- выбрать 10–15 секунд, где интерфейс хорошо движется;
- убрать начало/конец записи и лишние действия;
- пока оставить без голоса;
- добавить временную подпись:

```text
Java Swing real-time visualization prototype
```

### Блок B — место для первого коммита

Сразу после блока A поставить пустой placeholder длительностью 15–20 секунд:

```text
[BASELINE b0658a5 — запуск и первый экран]
```

### Блок C — место для кода

Следующий placeholder длительностью 30–45 секунд:

```text
[200 × 800 matrix / Timer / arraycopy / paint loop]
```

### Блок D — место для нагрузки

Placeholder:

```text
[LOAD BASELINE — commit pending]
```

### Блок E — место для измерений и результата

Placeholder:

```text
[MEASUREMENTS]
[CHANGE]
[BEFORE / AFTER]
```

Так timeline существует уже сейчас, даже если большая его часть пока состоит из пустых карточек.

## Ритм дальнейшей работы

Каждый законченный технический шаг закрывается одним 20–40-минутным контентным хвостом:

1. Переименовать и положить запись в правильный каталог.
2. Добавить лучший фрагмент на timeline вместо placeholder.
3. Внести факт в `docs/experiment-notes.md`.
4. Добавить или исправить абзац в `docs/article-draft.md`.
5. Обновить сцену в `docs/video-storyboard.md`.
6. Экспортировать короткий черновик только после крупного смыслового блока.

Не нужно после каждого шага:

- записывать финальный голос;
- полировать переходы;
- экспортировать весь ролик;
- переписывать статью с начала.

## Голос

Голос записывается блоками по 20–60 секунд после того, как соответствующий фрагмент экрана уже стоит на timeline.

Правило:

> Не подгонять видеоряд под воображаемую речь. Сначала увидеть конкретные кадры, затем объяснить именно их.

Допустимы:

- естественные паузы;
- спокойная подача;
- отдельные перезаписи одной фразы;
- небольшая импровизация относительно текста.

Не нужно записывать весь голос одним дублем.

## Монтажный проход

### Проход 1 — смысл

- правильный порядок сцен;
- нет ли недоказанных утверждений;
- виден ли код;
- понятен ли переход между версиями.

### Проход 2 — темп

- убрать ожидание запуска;
- убрать повторения;
- сократить длинные перемещения мыши;
- оставить паузу после важных цифр.

### Проход 3 — звук и подписи

- выровнять громкость речи;
- убрать очевидный фоновый шум;
- добавить только необходимые подписи;
- проверить термины и цифры.

После третьего прохода ролик можно выпускать. Четвёртый проход разрешён только для исправления конкретной проблемы.

---

## Защита от выгорания

- Максимум одна техническая и одна контентная задача за сессию.
- Не пересматривать весь материал после каждого нового файла.
- Не улучшать уже понятную сцену, пока timeline содержит пустые placeholders.
- Не исправлять баг, который не мешает текущей истории.
- Остановиться, когда получен артефакт с заранее заданным именем.
- Черновой монтаж должен выглядеть как черновой монтаж. Это нормально.

Красная линия:

> Если на одну минуту готового видео ушло больше двух часов монтажа, сначала упростить сцену, а не выделять ещё больше сил.
